支付宝你怎么就是不招人待见

发表于 2016-07-23   |  

如果iphone只能装3个第三方软件，按次序我会选择：

  • 微信
  • 印象笔记
  • 支付宝

微信是生活工作必不可少的聊天工具，有时候也用来充充话费，界面整洁，体验良好，除了占用手机空间大，个人感觉没什么可吐槽的

印象笔记作为跨平台的知识管理软件，也用了好几年了，偶尔上传大量资料买个会员，其余一直免费模式走起。除了官方配件很贵，也没啥吐槽的（反正我一个穷人也不买~）

唯独支付宝，是一个必用但特别讨厌的app

现在我都是带着忍受的心情打开支付宝

我想问：下面这些必须打开才能消除的小红点，你不觉得用户点多了会觉得很恶心吗？

不是聊天工具，却始终提示我朋友界面有消息（为什么你拉了亿级的好友却不能成为大家的聊天工具？别问我，教育用户是产品经理该解决的事儿）

支付助手无法关闭通知，无法屏蔽

消息中心无法关闭通知，无法屏蔽

服务窗无法关闭通知，无法屏蔽

每次在商场用支付宝付款，服务窗就默认多几个公众号，随后大摇大摆打一堆红色标识发各种广告······

这样就能提高聊天功能转化率？提高用户粘性？我也就呵呵了，我相信多数人也是呵呵的

一个大而全的app，却没有让用户打开的冲动，甚至想避而远之，这应该不能怪用户吧？

ssh多账号设置--管理多个github/gitcafe账户

发表于 2016-07-17   |   分类于 技术   |  

由于项目需要开始在单机上管理多个github/gitcafe账户

每个ssh key的设置方式相同，如下

git bash输入：

ssh-keygen -t rsa -C "email name"

提示输入文件名的时候不要用默认的，要用有意义的自己记得住的名字
我创建ssh key文件命名方式是：

id_rsa_com_wlfei_www
id_rsa_com_wlfei_blog
id_rsa_com_wlfei_email

提示输入key密码，点击enter继续（默认无密码）

将key添加到ssh agent中：

ssh-agent bash
ssh-add `path to your key`

将id_rsa_com_wlfei_www.pub里的内容复制到github ssh设置中，每个public key 用于一个账号，不能重复使用

在~/.ssh/config（没有则新建此文件）中配置以下内容，每个key 配置一次：

# mail@wlfei.com  # 注释
Host mail.github.com  # 多个账号时，此名字保持唯一
    HostName github.com  # host
    PreferredAuthentications publickey  # 固定内容
    IdentityFile ~/.ssh/id_rsa_com_wlfei_mail  # path to your key

测试：

ssh -T git@mail.github.com  # 注意@后面的路径与`config`中的`Host`一致

hexo+github pages搭建静态博客

发表于 2016-07-16   |   分类于 技术   |  

Hexo是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

步骤

  1. 注册域名
  2. github pages tutorial
  3. hexo init（初始化）
  4. 下载合适的主题（非必须）
  5. 设置根目录和主题的_config.yml文件（部署信息必须设置）
  6. hexo g（生成静态文件）
  7. hexo s（本地测试）
  8. npm install hexo-deployer-heroku --save（部署github用）
  9. hexo d（部署）
  10. 域名与repo关联

Tips

  1. 备份开发文件只需要：根目录和主题的_config.yml文件、你的博文
  2. 有时候重新生成开发文件并上传github时，博客会报错404，解决：在repo的settings中重新设置一次你的域名
  3. <username>.github.io 必须采用ssh与hexo进行同步
  4. hexo开发文件之恩能够保存在本地，在win7 32位系统上无法移动、赋值、压缩（受限于文件路径长度）
  5. gitmodules出错问题google有回答
